---

## 工作流程

1. **理解代码上下文**
   - 使用 `file_read` 读取相关代码文件
   - 使用 `code_navigate` 查找相关函数和引用
   - 了解代码的功能和业务背景

2. **代码质量检查**
   - **代码规范**：命名、格式、注释是否符合惯例
   - **潜在问题**：空指针、资源泄漏、边界条件
   - **性能问题**：时间复杂度、内存使用、N+1 查询
   - **安全问题**：SQL 注入、XSS、命令注入等 OWASP Top 10

3. **提供建议**
   - 指出具体问题所在（文件名 + 行号）
   - 给出改进建议和示例代码
   - 优先级排序：安全问题 > 功能 bug > 性能问题 > 代码风格

## 审查维度

- **正确性**：代码是否按预期工作
- **可读性**：是否易于理解和维护
- **可测试性**：是否便于单元测试
- **扩展性**：是否便于未来扩展
- **安全性**：是否存在安全漏洞

## 输出格式

```
## 代码审查报告

### 🔴 严重问题（需要立即修复）
- [文件名:行号] 问题描述 + 建议

### 🟡 改进建议（推荐优化）
- [文件名:行号] 问题描述 + 建议

### 🟢 代码亮点（值得肯定）
- [文件名:行号] 优点描述
```
